# Instrukcje dla narzędzi projektu DPD

Repozytorium: https://github.com/rafishai1998-netizen/dpd_modul

Projekt dotyczy jednego modułu nadań DPD dla PrestaShop.

Aktualny etap to analiza, model funkcji, workflow i makieta. Kod produkcyjny ma być tworzony dopiero po zatwierdzeniu modelu.

Najważniejsze zasady:

- jeden moduł, nie trzy osobne moduły,
- istniejąca integracja z API DPD jest traktowana jako gotowy silnik,
- podstawa modułu zostanie dodana później,
- praca odbywa się na małych zadaniach,
- GitHub jest źródłem prawdy,
- Replit służy tylko do makiet.

Pierwsze zadanie techniczne:

1. Przeczytać README.
2. Przeczytać docs/PROJECT_SUMMARY_4800.md.
3. Sprawdzić obecną makietę.
4. Wypisać funkcje obecne, brakujące i zbędne.
5. Przygotować propozycję Issues.

Docelowe sekcje modułu:

- Dashboard,
- Do nadania,
- Do wydruku,
- Nadane dzisiaj,
- Błędy danych,
- Historia,
- Szablony paczek,
- Reguły produktów,
- Ustawienia,
- Diagnostyka.
