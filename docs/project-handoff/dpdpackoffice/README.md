# DPD Pack Office — pakiet przekazania projektu

Ten pakiet służy do przekazania projektu pomiędzy:

- ChatGPT — analiza, architektura, planowanie, audyt i przygotowanie promptów,
- Codex — praca na repozytorium, zmiany plików, testy i raport,
- Replit — szybkie prototypowanie, podgląd interfejsu i eksperymenty,
- GitHub — źródło prawdy dla kodu, dokumentacji, decyzji i historii zmian.

## Najważniejsze pliki

1. `01_start/START_HERE.md` — punkt wejścia.
2. `02_mockup/dpdpackoffice-makieta-v4.html` — samodzielna makieta HTML.
3. `03_prompts/CHATGPT_MASTER_PROMPT.md` — główny prompt do nowego czatu.
4. `03_prompts/CODEX_IMPLEMENTATION_PROMPT.md` — prompt wykonawczy dla Codex.
5. `03_prompts/REPLIT_PROTOTYPE_PROMPT.md` — prompt dla Replit.
6. `04_docs/PROJECT_BRIEF.md` — cel, zakres i architektura.
7. `04_docs/ACCEPTANCE_CRITERIA.md` — kryteria akceptacji.
8. `04_docs/TEST_PLAN.md` — plan testów.
9. `04_docs/RISKS_AND_SECURITY.md` — ryzyka i bezpieczeństwo.
10. `06_repo/REPOSITORY_STRUCTURE.md` — docelowa struktura repozytorium.

## Status

Pakiet zawiera makietę i pełną dokumentację startową. Nie zawiera gotowego modułu PrestaShop ani integracji z API DPD.

## Zasada nadrzędna

Nie wolno uznać projektu za ukończony bez:

- repozytorium z kodem,
- wyników testów,
- listy zmienionych plików,
- potwierdzenia kryteriów akceptacji,
- raportu ograniczeń i ryzyk.
