## Co zrobić po otrzymaniu tego pakietu

1. Rozpakuj ZIP.
2. Otwórz `02_mockup/dpdpackoffice-makieta-v4.html`.
3. Przeczytaj `04_docs/PROJECT_BRIEF.md`.
4. Załóż lub wskaż repozytorium GitHub.
5. Umieść cały pakiet w katalogu:
   `docs/project-handoff/dpdpackoffice/`
6. W nowym czacie ChatGPT użyj:
   `03_prompts/CHATGPT_MASTER_PROMPT.md`
7. Po zatwierdzeniu planu użyj:
   `03_prompts/CODEX_IMPLEMENTATION_PROMPT.md`
8. Replit używaj do:
   - podglądu interfejsu,
   - szybkich prototypów,
   - testów UX,
   - nie jako jedynego źródła prawdy.
9. Wszystkie trwałe zmiany zapisuj w GitHub.

## Kolejność pracy

1. Audyt repozytorium.
2. Inwentaryzacja istniejących modułów DPD.
3. Decyzja: naprawa, refaktor minimalny albo nowa implementacja.
4. Implementacja etapami.
5. Testy statyczne.
6. Testy instalacji.
7. Testy Back Office.
8. Testy błędów i bezpieczeństwa.
9. Budowa ZIP.
10. Raport końcowy.

## Zakaz

Nie tworzyć kolejnej wersji modułu bez sprawdzenia, czy podobny kod lub dokument już istnieje.
