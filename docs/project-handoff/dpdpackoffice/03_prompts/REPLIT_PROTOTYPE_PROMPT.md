# PROMPT DLA REPLIT

Pracujemy nad makietą panelu Back Office `dpdpackoffice`.

Punkt wejścia:
`02_mockup/dpdpackoffice-makieta-v4.html`

Cel:
- uruchomić makietę,
- poprawić czytelność,
- zachować zakres funkcjonalny,
- nie zmieniać logiki biznesowej bez uzgodnienia,
- przygotować wariant desktop i mobile,
- zachować działanie offline.

Wymagania:
- jeden samodzielny plik HTML albo jasno opisana struktura,
- brak zewnętrznych bibliotek, jeśli nie są konieczne,
- brak danych wrażliwych,
- brak połączeń z prawdziwym API,
- czytelne statusy i ostrzeżenia,
- stabilny układ tabeli produktów,
- masowe zaznaczanie tylko listów z numerem listu,
- niewydrukowane listy na górze,
- ostrzeżenie przed ponownym drukiem,
- ostrzeżenie przed ponownym nadaniem.

Po pracy przygotuj:
- listę zmian,
- zrzuty lub opis widoków,
- wskazanie pliku wejściowego,
- wskazanie pliku końcowego,
- ograniczenia,
- rekomendacje dla implementacji PrestaShop.
