# DECYZJE PROJEKTOWE

## D-001
`dpdpackoffice` jest osobnym modułem Back Office.

## D-002
`dpdpackoffice` nie zastępuje `dpdshipmvp`.

## D-003
Brak automatycznych wywołań API po otwarciu panelu.

## D-004
Nie modyfikujemy core PrestaShop.

## D-005
Repozytorium GitHub jest źródłem prawdy.

## D-006
Replit może służyć do prototypów, ale wynik musi zostać przeniesiony i zweryfikowany w repozytorium.

## D-007
Codex wykonuje zmiany dopiero po audycie repozytorium.

## D-008
Nie uznajemy deklaracji „100% ukończenia” bez testów i dowodów.

## D-009
Makieta HTML jest źródłem wymagań UX, ale nie źródłem logiki biznesowej.

## D-010
Każdy ZIP musi przejść walidację struktury.
